## Endpoint

```
POST /api/flashcards
```

Creates a new flashcard with front and back content.

## Request

### Headers

<ParamField header="Content-Type" type="string" required>
  Must be set to `application/json`
</ParamField>

### Body Parameters

<ParamField body="front" type="string" required>
  The text to display on the front of the flashcard (the question or prompt)
</ParamField>

<ParamField body="back" type="string" required>
  The text to display on the back of the flashcard (the answer or explanation)
</ParamField>

### Request Example

<CodeGroup>

```bash cURL
curl -X POST http://localhost:3000/api/flashcards \
  -H "Content-Type: application/json" \
  -d '{
    "front": "What is the capital of France?",
    "back": "Paris"
  }'
```

```javascript JavaScript
fetch('http://localhost:3000/api/flashcards', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    front: 'What is the capital of France?',
    back: 'Paris'
  })
})
  .then(response => response.json())
  .then(data => console.log(data));
```

```python Python
import requests

url = 'http://localhost:3000/api/flashcards'
data = {
    'front': 'What is the capital of France?',
    'back': 'Paris'
}

response = requests.post(url, json=data)
print(response.json())
```

</CodeGroup>

## Response

### Success Response (200 OK)

<ResponseField name="message" type="string">
  Confirmation message indicating the flashcard was saved successfully
</ResponseField>

**Example Response:**

```json
{
  "message": "Flashcard saved successfully"
}
```

### Error Responses

#### 405 Method Not Allowed

Returned when using an HTTP method other than POST.

**Headers:**
```
Allow: POST
```

**Response Body:**
```
Method GET Not Allowed
```

## Implementation Notes

<Note>
  The current implementation logs the request body to the console for debugging purposes. In production, flashcard data should be persisted to a database.
</Note>

<Warning>
  This endpoint does not currently validate the request body. Ensure that both `front` and `back` fields are provided in your requests.
</Warning>

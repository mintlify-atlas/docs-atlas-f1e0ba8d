## Get Up and Running

This guide will walk you through setting up the Flashcard App on your local machine and creating your first flashcard.

<Note>
Make sure you have **Node.js** (version 18 or higher) and **npm** installed on your machine before proceeding.
</Note>

## Prerequisites

Before you begin, ensure you have:

- Node.js 18+ installed
- npm, yarn, pnpm, or bun package manager
- A code editor (VS Code recommended)
- Terminal/command line access

<Steps>
  <Step title="Clone and Install Dependencies">
    First, navigate to the flashcard-app directory and install all required dependencies:

    <CodeGroup>
      ```bash npm
      cd flashcard-app
      npm install
      ```

      ```bash yarn
      cd flashcard-app
      yarn install
      ```

      ```bash pnpm
      cd flashcard-app
      pnpm install
      ```

      ```bash bun
      cd flashcard-app
      bun install
      ```
    </CodeGroup>

    This will install all dependencies including:
    - Next.js 14.2.3
    - React 18
    - Chakra UI 2.8.2
    - Framer Motion
    - Tailwind CSS
    - And more...

    <Note>
    The installation may take a few minutes depending on your internet connection.
    </Note>
  </Step>

  <Step title="Start the Development Server">
    Once dependencies are installed, start the Next.js development server:

    <CodeGroup>
      ```bash npm
      npm run dev
      ```

      ```bash yarn
      yarn dev
      ```

      ```bash pnpm
      pnpm dev
      ```

      ```bash bun
      bun dev
      ```
    </CodeGroup>

    You should see output similar to:

    ```bash
    ▲ Next.js 14.2.3
    - Local:        http://localhost:3000
    - Network:      http://192.168.1.x:3000

    ✓ Ready in 2.5s
    ```

    Open your browser and navigate to **http://localhost:3000** to see the app running.

    <Note>
    The development server supports hot reloading - any changes you make to the code will automatically refresh the page.
    </Note>
  </Step>

  <Step title="Create Your First Flashcard">
    Now let's create your first flashcard!

    ### Navigate to the Create Page

    1. In the navigation bar at the top, click on **"Create Flashcard"**
    2. Or navigate directly to http://localhost:3000/create

    ### Fill Out the Form

    You'll see a form with two input fields:

    ```javascript
    <Input
      placeholder="Front of the flashcard"
      value={front}
      onChange={(e) => setFront(e.target.value)}
    />
    <Input
      placeholder="Back of the flashcard"
      value={back}
      onChange={(e) => setBack(e.target.value)}
    />
    ```

    3. Enter the **front** content (e.g., "What is React?")
    4. Enter the **back** content (e.g., "A JavaScript library for building user interfaces")

    ### Save the Flashcard

    5. Click the **"Save Flashcard"** button

    The flashcard data will be logged to the browser console. You can view it by:
    - Right-click on the page
    - Select **"Inspect"**
    - Click the **"Console"** tab

    You should see:
    ```
    Front: What is React?
    Back: A JavaScript library for building user interfaces
    ```

    <Note>
    Currently, flashcards are logged to the console. In future versions, they will be saved to a database for persistent storage.
    </Note>
  </Step>

  <Step title="View Your Flashcards">
    Navigate back to the home page to view all flashcards:

    1. Click **"All flashcards"** in the navigation bar
    2. Or go to http://localhost:3000

    You'll see the flashcard collection displayed:

    ```javascript pages/index.js
    const flashcards = [
      { id: 1, front: 'test1', back: 'test2' },
      { id: 2, front: 'test3', back: 'test4' },
      { id: 3, front: 'test5', back: 'test6' },
    ];
    ```

    Each flashcard shows:
    - **Front:** The question or term
    - **Back:** The answer or definition

    <Note>
    The current version displays example flashcards. Database integration is planned for future releases.
    </Note>
  </Step>
</Steps>

## Project Structure

Here's an overview of the key files and directories:

```
flashcard-app/
├── pages/
│   ├── index.js          # Home page - displays all flashcards
│   ├── create.js         # Create flashcard page
│   └── api/
│       └── flashcards.js # API route for flashcard operations
├── components/
│   ├── layout.js         # Main layout wrapper
│   └── Navigation.js     # Navigation bar component
├── app/
│   ├── layout.js         # Root layout with metadata
│   └── page.js           # App directory default page
├── public/               # Static assets
├── package.json          # Dependencies and scripts
└── next.config.mjs       # Next.js configuration
```

## Available Scripts

The app comes with several npm scripts defined in `package.json`:

```json
{
  "scripts": {
    "dev": "next dev",      // Start development server
    "build": "next build",  // Build for production
    "start": "next start",  // Start production server
    "lint": "next lint"     // Run ESLint
  }
}
```

### Development
```bash
npm run dev
```
Starts the development server with hot reloading at http://localhost:3000

### Production Build
```bash
npm run build
npm start
```
Builds and starts the optimized production version

### Linting
```bash
npm run lint
```
Runs ESLint to check code quality

## Troubleshooting

<AccordionGroup>
  <Accordion title="Port 3000 is already in use">
    If port 3000 is already occupied, Next.js will automatically try the next available port (3001, 3002, etc.). You can also specify a custom port:
    
    ```bash
    npm run dev -- -p 3001
    ```
  </Accordion>

  <Accordion title="Module not found errors">
    If you see module not found errors, try deleting `node_modules` and reinstalling:
    
    ```bash
    rm -rf node_modules package-lock.json
    npm install
    ```
  </Accordion>

  <Accordion title="Styles not loading">
    Make sure both Chakra UI and Tailwind CSS are properly configured. Check that `tailwind.config.js` and `postcss.config.mjs` exist in the root directory.
  </Accordion>
</AccordionGroup>

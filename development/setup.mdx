## Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js** (version 18 or higher)
- **npm** (comes with Node.js) or your preferred package manager
- A code editor (VS Code recommended)

## Installation

<Steps>
  <Step title="Clone the repository">
    Clone the Flashcard App repository to your local machine:
    
    ```bash
    git clone <repository-url>
    cd flashcard-app
    ```
  </Step>

  <Step title="Install dependencies">
    Install all required dependencies using npm:
    
    ```bash
    npm install
    ```
    
    This will install:
    - **Next.js 14.2.3** - React framework
    - **React 18** - UI library
    - **Chakra UI 2.8.2** - Component library
    - **Tailwind CSS 3.4.1** - Utility-first CSS framework
    - **Framer Motion 11.1.9** - Animation library
    - **ESLint** - Code linting
  </Step>

  <Step title="Start the development server">
    Run the development server:
    
    ```bash
    npm run dev
    ```
    
    The app will be available at [http://localhost:3000](http://localhost:3000)
  </Step>
</Steps>

## Available Scripts

The project includes the following npm scripts defined in `package.json`:

<CodeGroup>
```bash Development
npm run dev
```

```bash Production Build
npm run build
```

```bash Start Production Server
npm start
```

```bash Linting
npm run lint
```
</CodeGroup>

### Script Details

| Script | Command | Description |
|--------|---------|-------------|
| `dev` | `next dev` | Starts the development server with hot reload |
| `build` | `next build` | Creates an optimized production build |
| `start` | `next start` | Runs the production server |
| `lint` | `next lint` | Runs ESLint to check code quality |

<Note>
  The development server supports hot module replacement (HMR), so changes you make will be reflected immediately without refreshing the page.
</Note>

## Project Structure

The Flashcard App follows the Next.js 14 App Router structure:

```
flashcard-app/
├── app/
│   ├── layout.js          # Root layout with providers
│   ├── page.js            # Home page
│   └── globals.css        # Global styles
├── components/            # React components
├── public/               # Static assets
├── next.config.mjs       # Next.js configuration
├── tailwind.config.js    # Tailwind CSS configuration
└── package.json          # Dependencies and scripts
```

## Development Workflow

<Steps>
  <Step title="Make changes">
    Edit files in the `app/` or `components/` directories. The development server will automatically reload.
  </Step>

  <Step title="Check for errors">
    Run the linter to catch potential issues:
    
    ```bash
    npm run lint
    ```
  </Step>

  <Step title="Test the build">
    Before deploying, ensure the production build works:
    
    ```bash
    npm run build
    npm start
    ```
  </Step>
</Steps>

<Tip>
  Use the ESLint extension in VS Code for real-time linting feedback as you code.
</Tip>

## Environment Variables

If your app requires environment variables, create a `.env.local` file in the root directory:

```bash .env.local
# Add your environment variables here
NEXT_PUBLIC_API_URL=your-api-url
```

<Warning>
  Never commit `.env.local` or any files containing secrets to version control.
</Warning>

## Troubleshooting

### Port Already in Use

If port 3000 is already in use, you can specify a different port:

```bash
npm run dev -- -p 3001
```

### Dependencies Issues

If you encounter dependency conflicts, try deleting `node_modules` and reinstalling:

```bash
rm -rf node_modules package-lock.json
npm install
```

### Build Errors

Clear the Next.js cache if you experience build issues:

```bash
rm -rf .next
npm run build
```

## Overview

The Flashcard App is built with Next.js 14, which can be deployed to various platforms. This guide covers deployment to Vercel (recommended) and other hosting options.

## Deploy to Vercel (Recommended)

Vercel is the easiest and recommended way to deploy Next.js applications, created by the same team behind Next.js.

<Steps>
  <Step title="Sign up for Vercel">
    Create a free account at [vercel.com](https://vercel.com)
  </Step>

  <Step title="Import your repository">
    Click "Add New Project" and import your Git repository (GitHub, GitLab, or Bitbucket)
  </Step>

  <Step title="Configure project settings">
    Vercel auto-detects Next.js projects. Verify the settings:
    
    - **Framework Preset:** Next.js
    - **Build Command:** `npm run build`
    - **Output Directory:** `.next`
    - **Install Command:** `npm install`
  </Step>

  <Step title="Add environment variables (if needed)">
    If your app uses environment variables, add them in the project settings.
  </Step>

  <Step title="Deploy">
    Click "Deploy" and Vercel will build and deploy your app automatically.
  </Step>
</Steps>

<Note>
  Every push to your main branch will automatically trigger a new deployment.
</Note>

### Vercel CLI Deployment

You can also deploy using the Vercel CLI:

```bash
# Install Vercel CLI globally
npm install -g vercel

# Login to Vercel
vercel login

# Deploy to production
vercel --prod
```

## Deploy to Other Platforms

### Node.js Server

For deployment on a custom Node.js server:

<Steps>
  <Step title="Build the application">
    ```bash
    npm run build
    ```
  </Step>

  <Step title="Start the production server">
    ```bash
    npm start
    ```
    
    The app will run on port 3000 by default.
  </Step>

  <Step title="Configure environment">
    Set the `NODE_ENV` environment variable:
    
    ```bash
    NODE_ENV=production npm start
    ```
  </Step>
</Steps>

<Tip>
  Use a process manager like PM2 to keep your app running:
  
  ```bash
  npm install -g pm2
  pm2 start npm --name "flashcard-app" -- start
  pm2 save
  pm2 startup
  ```
</Tip>

### Docker Deployment

Create a `Dockerfile` in the project root:

```dockerfile Dockerfile
FROM node:18-alpine AS deps
RUN apk add --no-cache libc6-compat
WORKDIR /app

COPY package.json package-lock.json ./
RUN npm ci

FROM node:18-alpine AS builder
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY . .

RUN npm run build

FROM node:18-alpine AS runner
WORKDIR /app

ENV NODE_ENV production

RUN addgroup --system --gid 1001 nodejs
RUN adduser --system --uid 1001 nextjs

COPY --from=builder /app/public ./public
COPY --from=builder --chown=nextjs:nodejs /app/.next/standalone ./
COPY --from=builder --chown=nextjs:nodejs /app/.next/static ./.next/static

USER nextjs

EXPOSE 3000

ENV PORT 3000

CMD ["node", "server.js"]
```

Update `next.config.mjs` for standalone output:

```javascript next.config.mjs
/** @type {import('next').NextConfig} */
const nextConfig = {
  output: 'standalone',
};

export default nextConfig;
```

Build and run the Docker container:

```bash
# Build the image
docker build -t flashcard-app .

# Run the container
docker run -p 3000:3000 flashcard-app
```

### Static Export

<Warning>
  Static export disables some Next.js features like API routes, server-side rendering, and image optimization.
</Warning>

For static hosting (Netlify, GitHub Pages, AWS S3):

1. Update `next.config.mjs`:

```javascript next.config.mjs
/** @type {import('next').NextConfig} */
const nextConfig = {
  output: 'export',
};

export default nextConfig;
```

2. Build the static site:

```bash
npm run build
```

3. The static files will be in the `out/` directory.

## Environment Variables

### Development Environment

Create `.env.local` for local development:

```bash .env.local
NEXT_PUBLIC_API_URL=http://localhost:3000/api
```

### Production Environment

For production, set environment variables in your hosting platform:

<CodeGroup>
```bash Vercel
# In Vercel dashboard or via CLI
vercel env add NEXT_PUBLIC_API_URL production
```

```bash Node.js
# Set before running the app
export NEXT_PUBLIC_API_URL=https://api.example.com
npm start
```

```dockerfile Docker
# In Dockerfile
ENV NEXT_PUBLIC_API_URL=https://api.example.com
```
</CodeGroup>

<Note>
  Variables prefixed with `NEXT_PUBLIC_` are exposed to the browser. Never use this prefix for secrets.
</Note>

## Performance Optimization

### Build Optimization

The production build automatically:
- Minifies JavaScript and CSS
- Optimizes images
- Generates static pages where possible
- Tree-shakes unused code

### Caching Strategy

Next.js uses aggressive caching:
- Static assets are cached indefinitely
- Pages are incrementally regenerated
- API routes can use custom cache headers

### Image Optimization

Use the Next.js `<Image>` component for automatic optimization:

```jsx
import Image from 'next/image';

<Image 
  src="/flashcard.png" 
  alt="Flashcard" 
  width={500} 
  height={300}
  priority
/>
```

## Pre-deployment Checklist

<AccordionGroup>
  <Accordion title="Code Quality">
    - [ ] Run `npm run lint` and fix all errors
    - [ ] Remove console.log statements
    - [ ] Check for TypeScript errors (if using TypeScript)
    - [ ] Test all features in development mode
  </Accordion>

  <Accordion title="Build Testing">
    - [ ] Run `npm run build` successfully
    - [ ] Test with `npm start` locally
    - [ ] Check for build warnings
    - [ ] Verify bundle size is reasonable
  </Accordion>

  <Accordion title="Configuration">
    - [ ] Set all required environment variables
    - [ ] Update `next.config.mjs` if needed
    - [ ] Configure domain and SSL certificate
    - [ ] Set up analytics (if required)
  </Accordion>

  <Accordion title="Security">
    - [ ] Review environment variables (no secrets in client-side)
    - [ ] Enable security headers in `next.config.mjs`
    - [ ] Set up proper CORS policies
    - [ ] Audit dependencies with `npm audit`
  </Accordion>
</AccordionGroup>

## Post-deployment

### Monitoring

Vercel provides built-in analytics and monitoring. For other platforms:

- Set up error tracking (Sentry, LogRocket)
- Configure performance monitoring
- Set up uptime monitoring
- Enable logging

### Continuous Deployment

With Vercel, every Git push automatically deploys:
- **Main branch** → Production
- **Other branches** → Preview deployments

### Custom Domains

<Steps>
  <Step title="Add domain in Vercel">
    Go to Project Settings → Domains and add your custom domain
  </Step>

  <Step title="Configure DNS">
    Add the provided DNS records to your domain registrar
  </Step>

  <Step title="Wait for propagation">
    SSL certificates are automatically provisioned (5-10 minutes)
  </Step>
</Steps>

## Troubleshooting

### Build Failures

**Issue:** Build fails with dependency errors

```bash
# Clear cache and reinstall
rm -rf .next node_modules package-lock.json
npm install
npm run build
```

### Runtime Errors

**Issue:** App works locally but fails in production

- Check environment variables are set correctly
- Verify `NODE_ENV=production`
- Review build logs for warnings
- Test production build locally with `npm run build && npm start`

### Performance Issues

**Issue:** Slow page loads

- Enable Next.js Analytics in Vercel
- Check bundle size with `npm run build`
- Optimize images and assets
- Implement lazy loading for components

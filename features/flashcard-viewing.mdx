## Overview

The flashcard viewing feature displays all available flashcards in a list format. This is implemented in the home page (`pages/index.js`) and serves as the main entry point for users to see their flashcard collection.

## Component Implementation

The home page component renders flashcards in a simple list structure:

<CodeGroup>

```javascript pages/index.js
import React from 'react';
import Layout from '../components/layout';

const HomePage = () => {
  // Example flashcards data
  const flashcards = [
    { id: 1, front: 'test1', back: 'test2' },
    { id: 2, front: 'test3', back: 'test4' },
    { id: 3, front: 'test5', back: 'test6' },
  ];

  return (
    <Layout>
      <div>
        <h1>All Flashcards</h1>
        <ul>
          {flashcards.map((card) => (
            <li key={card.id}>
              <div>
                <strong>Front:</strong> {card.front}
              </div>
              <div>
                <strong>Back:</strong> {card.back}
              </div>
            </li>
          ))}
        </ul>
      </div>
    </Layout>
  );
};

export default HomePage;
```

</CodeGroup>

## Data Structure

Each flashcard object contains three key properties:

```javascript
{
  id: 1,           // Unique identifier
  front: 'test1',  // Question or prompt
  back: 'test2'    // Answer or explanation
}
```

<Note>
  Currently, the flashcards are stored as static data within the component. In a production environment, this would be replaced with data fetched from an API or database.
</Note>

## Rendering Logic

The component uses the `map()` method to iterate over flashcards and render each one:

```javascript
{flashcards.map((card) => (
  <li key={card.id}>
    <div>
      <strong>Front:</strong> {card.front}
    </div>
    <div>
      <strong>Back:</strong> {card.back}
    </div>
  </li>
))}
```

### Key Properties

- **key={card.id}**: React requires a unique key for list items to optimize rendering
- Each flashcard displays both front and back content simultaneously

## Display Pattern

The current implementation shows both sides of each flashcard simultaneously:

```text
Front: test1
Back: test2
```

This provides a quick reference view of all flashcards but differs from a typical study mode where answers are hidden until revealed.

## Technology Stack

The viewing feature leverages:

- **Next.js**: For server-side rendering and routing
- **React**: For component-based UI
- **Chakra UI**: For styling and layout components (Navigation, Flex, Box, Text)
- **Next.js Link**: For optimized client-side navigation

<Note>
  The app uses both Chakra UI and Tailwind CSS according to `package.json`, providing flexibility in styling approaches.
</Note>
